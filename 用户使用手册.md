## 系统概述

### 什么是太空梦想计划？

太空梦想计划是一个**深空AI生存模拟系统**，模拟在深空环境中维持生命支持系统的运行。系统包含：

- **实时监控系统**: 监控食物、医疗、能源、环境等关键指标
- **AI智能决策**: DeepSeek AI分析当前状态并提供优化建议
- **多系统联动**: 各子系统相互影响，形成复杂的生存生态
- **紧急响应**: 自动检测和应对各种危机情况
- **预测分析**: 基于当前状态预测未来生存趋势

### 核心概念

| 指标 | 说明 | 正常范围 |
|------|------|----------|
| **生存指数** | 综合生存能力评分 | 80-100% |
| **任务天数** | 已执行的任务天数 | - |
| **预计生存天数** | 基于当前资源的预估 | >30天 |
| **乘员数量** | 当前基地宇航员人数 | 1-20人 |

## 主控制台（仪表盘）

### 功能说明

主控制台是系统的**核心监控面板**，显示所有关键指标的实时状态。

### 显示内容

#### 1. 顶部状态栏
- **乘员数量**: 当前基地的宇航员总数
- **生存指数**: 综合评分（0-100%）
- **任务天数**: 已运行的天数

#### 2. 生存指数仪表盘
- 圆形仪表显示当前生存指数
- 颜色编码:
  - 🟢 绿色 (80-100%): 优秀
  - 🟡 黄色 (50-79%): 警告
  - 🔴 红色 (0-49%): 危险

#### 3. 雷达图
显示5个维度的状态:
- 食物稳定性
- 医疗安全性
- 能源水平
- 氧气浓度
- 水资源储备

#### 4. 预测时间线
- 显示未来30/60/90/120天的预测生存指数
- 基于当前衰减率计算
- 帮助规划长期策略

#### 5. 关键指标卡片
- **预计生存时间**: 基于资源消耗的动态计算
- **AI饮食建议**: 当前的营养配给策略
- **紧急模式指示**: 是否处于紧急状态

### 使用方法

**无需操作** - 数据每10秒自动刷新

### 联动效果

| 触发条件 | 联动效果 |
|---------|---------|
| 生存指数<50% | 仪表盘变红，触发警告动画 |
| 生存指数<30% | 进入紧急模式，全系统警报 |
| 资源耗尽 | 预计生存时间归零，触发危机协议 |
| 添加资源 | 生存指数上升，预测曲线改善 |

### 实用技巧

💡 **提示1**: 观察预测时间线的下降趋势，提前补充资源  
💡 **提示2**: 生存指数突然下降时，检查AI日志了解原因  
💡 **提示3**: 点击侧边栏其他模块可查看详细管理界面

## 食物资源管理

### 功能说明

管理基地的食物库存、消耗速率和保鲜系统。

### 页面布局

```
┌──────────────────────────────────────┐
│  添加食物表单                         │
│  [名称] [数量] [类型] [过期日期] [添加]│
├──────────────────────────────────────┤
│  消耗控制                             │
│  [速率滑块] [活动强度] [应用]         │
├──────────────────────────────────────┤
│  紧急配给模式                         │
│  [启用复选框] [百分比滑块] [切换]     │
├──────────────────────────────────────┤
│  保鲜控制                             │
│  [过期预警天数] [最低库存%] [应用]    │
├──────────────────────────────────────┤
│  库存列表                             │
│  - 蛋白质储备: XX%                    │
│  - 水资源: XX%                        │
│  - 冷冻食品: XX%                      │
│  - 脱水食品: XX%                      │
├──────────────────────────────────────┤
│  保鲜温度区域                         │
│  [冷冻区°C] [冷藏区°C] [深冷区°C]     │
└──────────────────────────────────────┘
```

### 详细功能

#### 1. 添加食物

**操作步骤**:
1. 输入食物名称（如："合成蛋白质块"）
2. 输入数量（单位：kg或份）
3. 选择营养类型:
   - 蛋白质
   - 碳水化合物
   - 脂肪
   - 维生素
4. （可选）输入过期日期
5. 点击"添加"按钮

**联动效果**:
- ✅ **食物稳定性提升**: 每10单位增加1%稳定性
- ✅ **生存指数上升**: 食物维度改善
- ✅ **预测曲线改善**: 预计生存时间延长
- ✅ **AI重新计算**: 更新资源消耗预测

**示例**:
```
添加 50kg 蛋白质 → 食物稳定性 +5% → 生存指数 +1%
```

#### 2. 消耗控制

**操作步骤**:
1. 调整"每日消耗速率"滑块（0.1-5.0）
2. 选择活动强度:
   - 低活动量: 消耗减少20%
   - 正常活动: 基准消耗
   - 高活动量: 消耗增加30%
3. 点击"应用"

**联动效果**:
- ⚠️ **消耗速率改变**: 影响食物/水/蛋白质的日消耗
- ⚠️ **预测更新**: 预计生存时间重新计算
- ⚠️ **乘员影响**: 高活动量加速资源耗尽

**建议**:
- 资源充足时: 正常活动
- 资源紧张时: 低活动量模式
- 紧急情况: 配合紧急配给使用

#### 3. 紧急配给模式

**操作步骤**:
1. 勾选"启用紧急配给"
2. 调整配给百分比（10-100%）
3. 点击"切换模式"

**联动效果**:
- 🔴 **消耗大幅降低**: 按百分比减少每日消耗
- 🔴 **乘员满意度下降**: 长期影响健康状态
- 🔴 **生存时间延长**: 短期应急有效

**使用场景**:
- 食物稳定性<30%时启用
- 等待补给期间
- 紧急情况下的资源保存

#### 4. 保鲜控制

**操作步骤**:
1. 设置"过期预警天数"（1-30天）
2. 设置"最低库存预警%"（5-50%）
3. 点击"应用预警设置"

**联动效果**:
- ℹ️ **预警触发**: 达到阈值时在AI日志中提醒
- ℹ️ **无直接影响**: 仅监控，不改变系统状态

#### 5. 保鲜温度区域

**操作步骤**:
1. 设置三个温区:
   - 冷冻区: -18°C（标准冷冻）
   - 冷藏区: 4°C（保鲜）
   - 深冷区: -70°C（生物样本）
2. 点击"应用温度设置"

**联动效果**:
- ⚡ **能源消耗变化**: 更低温度需要更多能源
- ⚡ **食物保鲜度**: 适当温度延长保质期
- ⚡ **医疗物品保存**: 深冷区用于疫苗/血浆

**建议配置**:
```
冷冻区: -18°C (平衡能耗和保鲜)
冷藏区: 4°C  (日常食物)
深冷区: -70°C (医疗专用)
```

### 实战案例

**场景**: 食物稳定性降至40%，预计只能维持20天

**操作步骤**:
1. 启用紧急配给模式，设置为60%
2. 将活动强度改为"低活动量"
3. 添加新的食物储备（如有）
4. 观察预测曲线是否改善

**预期效果**:
- 消耗速率降低至原来的36% (60% × 0.6)
- 预计生存时间从20天延长至55天
- 生存指数缓慢回升

## 医疗冷链管理

### 功能说明

管理医疗物品的存储、温度控制和优先级分配。

### 页面布局

```
┌──────────────────────────────────────┐
│  添加医疗物品                         │
│  [名称] [类型] [数量] [温度] [优先级] │
│  [添加]                               │
├──────────────────────────────────────┤
│  温度控制                             │
│  [最低温度°C] [最高温度°C] [应用]     │
├──────────────────────────────────────┤
│  医疗物品列表                         │
│  - 疫苗冷藏库: -70°C [删除]           │
│  - 血浆储存: -65°C [删除]             │
│  - 药品柜: 4°C [删除]                 │
└──────────────────────────────────────┘
```

### 详细功能

#### 1. 添加医疗物品

**操作步骤**:
1. 输入物品名称（如："新冠疫苗"）
2. 选择类型:
   - 疫苗
   - 药品
   - 生物样本
3. 输入数量
4. 输入存储温度（°C）
5. 选择优先级:
   - 低优先级
   - 正常
   - 高优先级
   - 紧急
6. 点击"添加"

**联动效果**:
- ✅ **医疗安全性提升**: 每5单位增加1%安全性
- ✅ **生存指数上升**: 医疗维度改善
- ✅ **能源需求增加**: 低温存储消耗能源
- ✅ **AI记录**: 更新医疗库存并重新计算预测

**优先级说明**:
| 优先级 | 用途 | 能源保障 |
|--------|------|---------|
| 紧急 | 救命药物/疫苗 | 最高，即使能源不足也保持 |
| 高 | 重要药品 | 优先保障 |
| 正常 | 常规药品 | 标准保障 |
| 低 | 备用物资 | 可降低精度节省能源 |

#### 2. 温度控制

**操作步骤**:
1. 设置最低温度（如：-80°C）
2. 设置最高温度（如：-60°C）
3. 点击"应用温度范围"

**联动效果**:
- ⚡ **温度偏离警报**: 超出范围时触发警告
- ⚡ **能源消耗**: 更低的温度需要更多制冷
- ⚡ **物品安全**: 温度不当会导致物品失效

**推荐配置**:
```
疫苗/血浆: -80°C 至 -60°C
药品: 2°C 至 8°C
生物样本: -196°C（液氮）
```

#### 3. 医疗物品列表

**功能**:
- 查看所有医疗物品
- 显示当前存储温度
- 显示优先级
- 可删除过期/无用物品

**删除操作**:
1. 点击物品旁的"删除"按钮
2. 输入删除原因（如："过期"、"已使用"）
3. 确认删除

**联动效果**:
- ⚠️ **医疗安全性下降**: 移除物品降低总安全性
- ⚠️ **能源需求减少**: 少一个物品需要制冷
- ⚠️ **AI记录**: 记录移除原因

### 实战案例

**场景**: 收到一批新冠疫苗，需要妥善存储

**操作步骤**:
1. 添加物品:
   - 名称: "新冠疫苗批次A"
   - 类型: 疫苗
   - 数量: 100剂
   - 温度: -70°C
   - 优先级: 紧急
2. 确认温度范围设置为 -80°C 至 -60°C
3. 观察医疗安全性提升

**预期效果**:
- 医疗安全性 +20% (100/5)
- 生存指数 +6% (医疗权重30%)
- 能源消耗略有增加
- AI标记为"紧急"优先级，确保能源不足时优先保障

## 能源管理系统

### 功能说明

管理基地的能源分配、充电策略和低电量应急响应。

### 页面布局

```
┌──────────────────────────────────────┐
│  能源分配                             │
│  医疗区: [====30%====] 30%            │
│  食物区: [====25%====] 25%            │
│  环境控制: [====25%====] 25%          │
│  其他系统: [====20%====] 20%          │
│  当前总和: 100%                       │
│  [应用分配]                           │
├──────────────────────────────────────┤
│  充电策略                             │
│  [太阳能充电时间: 8小时]              │
│  [备用电源启用条件: 30%]              │
│  [应用策略]                           │
├──────────────────────────────────────┤
│  低电量响应                           │
│  自动关机顺序: [文本框]               │
│  保留的核心功能:                      │
│  ☑ 生命支持  ☑ 通信系统              │
│  ☑ 医疗设备  ☐ 导航系统              │
│  [应用配置]                           │
└──────────────────────────────────────┘
```

### 详细功能

#### 1. 能源分配

**操作步骤**:
1. 调整四个区域的滑块:
   - 医疗区（建议：25-35%）
   - 食物区（建议：20-30%）
   - 环境控制（建议：20-30%）
   - 其他系统（建议：15-25%）
2. 确保总和为100%（红色表示错误）
3. 点击"应用分配"

**联动效果**:
- ⚡ **衰减率变化**: 分配越低，该区域衰减越快
- ⚡ **生存指数影响**: 关键区域分配不足会降低指数
- ⚡ **AI决策调整**: AI会根据分配比例优化资源调度

**计算公式**:
```
实际衰减率 = 基础衰减率 / 分配比例系数

例如:
医疗区分配30% → 衰减正常
医疗区分配15% → 衰减速度翻倍
```

**推荐配置**:
| 场景 | 医疗 | 食物 | 环境 | 其他 |
|------|------|------|------|------|
| 正常 | 30% | 25% | 25% | 20% |
| 医疗危机 | 40% | 20% | 20% | 20% |
| 能源紧张 | 25% | 20% | 20% | 35% |

#### 2. 充电策略

**操作步骤**:
1. 设置"太阳能充电时间"（0-24小时）
2. 设置"备用电源启用条件"（0-100%）
3. 点击"应用策略"

**联动效果**:
- 🔋 **能源恢复**: 太阳能充电时间内能源缓慢恢复
- 🔋 **备用电源**: 低于阈值时自动启用备用电池
- 🔋 **续航时间**: 影响backup_power_hours指标

**建议配置**:
```
太阳能充电时间: 8小时（模拟日照周期）
备用电源启用: 30%（预留缓冲）
```

#### 3. 低电量响应

**操作步骤**:
1. 输入自动关机顺序（文本描述）
   - 例如："非关键系统→照明→空调→生命支持"
2. 选择要保留的核心功能:
   - ☑ 生命支持（氧气、温度）
   - ☑ 通信系统（与地球联系）
   - ☑ 医疗设备（冷链、急救）
   - ☐ 导航系统（可选）
3. 点击"应用配置"

**联动效果**:
- 🚨 **自动保护**: 能源<10%时自动执行关机顺序
- 🚨 **关键功能保障**: 选中的功能不受影响
- 🚨 **AI日志记录**: 记录低电量事件和采取的措施

**关机顺序建议**:
```
1. 科研设备（非必需）
2. 娱乐系统（完全可关闭）
3. 部分照明（保留必要照明）
4. 空调精度降低（维持基本温度）
5. 生命支持（最后保障）
```

### 实战案例

**场景**: 能源水平降至25%，需要优化分配

**操作步骤**:
1. 查看当前各区域状态
2. 临时调整分配:
   - 医疗区: 35%（提升5%）
   - 食物区: 20%（降低5%）
   - 环境控制: 20%（降低5%）
   - 其他系统: 25%（提升5%）
3. 点击"应用分配"
4. 观察能源衰减率变化

**预期效果**:
- 医疗区衰减减缓，安全性稳定
- 食物区衰减加快，需密切监控
- 整体能源消耗优化
- AI可能建议进一步调整

**策略对比**:
| 策略 | 优先级 | 典型行为 |
|------|--------|---------|
| 生存优先 | 人员安全第一 | 不惜代价保障生命支持 |
| 资源保留 | 物资安全 | 最小化资源消耗 |
| 任务继续 | 任务目标 | 即使有风险也继续任务 |

#### 4. AI偏好配置

**操作步骤**:
1. 调整风险承受度滑块（0-100%）
   - 0%: 极度保守，绝不冒险
   - 50%: 平衡（推荐）
   - 100%: 极度激进，愿意冒险
2. 点击"应用偏好"

**联动效果**:
- 🎯 **AI决策风格**: 影响AI的建议和行动
- 🎯 **资源调度**: 高风险承受度可能尝试激进方案
- 🎯 **紧急响应**: 低风险承受度会更早触发警报

**风险承受度影响**:
| 承受度 | AI行为特点 |
|--------|-----------|
| 0-30% | 保守，早期预警，避免任何风险 |
| 30-70% | 平衡，权衡利弊后决策 |
| 70-100% | 激进，愿意冒险换取收益 |

### 实战案例

**场景**: 模拟太阳风暴袭击，测试不同策略

**操作步骤**:
1. 选择场景: "太阳风暴"
2. 设置风险概率: 80%
3. 第一次模拟 - 策略"生存优先":
   - 点击"运行模拟"
   - 观察AI建议: 可能建议关闭非必要系统，全力保障生命支持
   - 记录结果
4. 重置系统（或记住当前状态）
5. 第二次模拟 - 策略"资源保留":
   - 再次运行
   - 观察AI建议: 可能建议进入地下储藏，保护物资
   - 对比两次结果
6. 第三次模拟 - 策略"任务继续":
   - 观察AI如何平衡任务和生存

**预期效果**:
- 辐射水平飙升到50-95%

- 学习到策略选择的重要性

## 常见问题

### Q1: 生存指数为什么会下降？

**A**: 生存指数下降通常由以下原因导致:
1. **资源自然消耗**: 食物、水、能源每天自动减少
2. **乘员增加**: 更多人=更快消耗
3. **紧急事件**: 太阳风暴、泄漏等突发事件
4. **系统联动**: 某个子系统故障影响其他系统

**解决方法**:
- 补充资源（添加食物、医疗物品等）
- 优化能源分配
- 启用紧急配给模式
- 查看AI日志了解具体原因

---

### Q2: 如何延长预计生存时间？

**A**: 有多种方法:
1. **增加资源**: 添加食物、水、医疗物品
2. **减少消耗**: 
   - 降低活动强度
   - 启用紧急配给
   - 减少乘员数量
3. **优化分配**: 合理分配能源，减少浪费
4. **改善环境**: 维持适宜的环境参数，减少额外消耗

---

### Q3: AI建议看不懂怎么办？

**A**: 
1. **简化问题**: 在AI对话中输入具体问题，如"现在最紧急的问题是什么？"
2. **查看日志**: AI日志中有详细的决策记录
3. **调整自动化级别**: 改为"手动模式"，AI只给建议不执行
4. **降低风险承受度**: 让AI更保守，建议更容易理解

---

### Q4: 系统卡住或不响应怎么办？

**A**: 
1. **检查网络连接**: 确保能访问后端服务器
2. **刷新页面**: 按F5或Ctrl+R
3. **清除缓存**: Ctrl+Shift+Delete清除浏览器缓存
4. **重启服务**: 如果是本地运行，重启Flask服务
5. **查看控制台**: F12打开开发者工具，查看Console是否有错误

---

### Q5: 如何备份当前状态？

**A**: 
1. **自动生成**: 系统每5分钟自动保存到localStorage
2. **手动导出**: 
   - 打开浏览器控制台（F12）
   - 输入: `localStorage.getItem('spaceSurvivalState')`
   - 复制输出的JSON字符串
   - 保存到文本文件
3. **恢复状态**:
   - 在控制台输入: `localStorage.setItem('spaceSurvivalState', '粘贴JSON')`
   - 刷新页面

---

### Q6: API密钥在哪里配置？

**A**: 
1. **获取密钥**: 访问 https://platform.deepseek.com/ 注册并获取API Key
2. **配置方法**:
   - 方法1: 创建`.env`文件，写入`DEEPSEEK_API_KEY=你的密钥`
   - 方法2: 设置环境变量`DEEPSEEK_API_KEY`
   - 方法3: Vercel部署时在环境变量中配置
3. **验证**: 重启服务后，AI功能应该可用

---

### Q7: 如何让系统运行更快？

**A**: 
1. **降低刷新频率**: 改为10-30秒
2. **关闭不必要的标签页**: 减少浏览器负担
3. **清理AI日志**: 日志过多会影响性能
4. **减少乘员数量**: 更少乘员=更少计算
5. **使用现代浏览器**: Chrome/Edge性能最好

---

### Q8: 紧急模式下如何退出？

**A**: 
1. **自动退出**: 当生存指数恢复到50%以上时自动退出
2. **手动解决**: 
   - 补充资源提升生存指数
   - 修复导致紧急的问题
   - 等待AI自动调整
3. **强制退出**: 目前不支持手动强制退出，必须通过改善状态来退出

---

### Q9: 数据会丢失吗？

**A**: 
- **短期**: 数据保存在内存中，刷新页面不会丢失
- **长期**: localStorage保存最近的状态，但有以下限制:
  - 清除浏览器数据会丢失
  - 更换设备/浏览器会丢失
  - 建议定期手动备份重要状态
- **Vercel部署**: Serverless环境下，每次请求都是新实例，状态不持久
