# 深空AI生存冷链决策系统 - 功能联动验证清单

## ✅ 已实现的核心联动功能

### 1. AI 真实决策联动（GLM-4-AIR）
**触发条件**: 每3秒自动调用智谱 API
**联动效果**:
- ✓ AI 建议"降低冷却" → 食物保鲜度 -2.5%，能源 +3%
- ✓ AI 建议"医疗优先" → 医疗安全 +3%，能源 -2%
- ✓ AI 建议"营养优化" → 蛋白质消耗优化，启动 AI 配给模式
- ✓ AI 检测到危险 → 插入红色预警日志

**验证方法**: 查看右侧日志区，会显示：
```
> AI 决策: [GLM-4 生成的建议]
> ✓ 已降低非关键区域冷却精度
> ✓ 已提升医疗冷链优先级
```

---

### 2. 能源联动系统
**触发条件**: 能源水平 < 40%
**联动效果**:
- 自动降低次级冷却系统精度
- 食物保鲜度加速衰减（衰减率 = (40 - 当前能源) × 0.1%）
- 日志显示："能源联动：已降低次级冷却系统精度X%"

**验证方法**: 
1. 等待能源自然衰减到 40% 以下
2. 观察日志区出现红色"能源联动"消息
3. 切换到"食物资源"视图，看到食物稳定性加速下降

---

### 3. 辐射联动系统
**触发条件**: 辐射等级 > 50
**联动效果**:
- 自动启动地下储藏模式
- 医疗区防护提升（防护值 = (辐射 - 50) × 0.05%）
- 牺牲部分食物保鲜度来保障医疗资源
- 日志显示："辐射联动：已启动地下储藏模式，医疗区防护+X%"

**验证方法**:
1. 等待随机触发的太阳风暴事件（5% 概率）
2. UI 切换为红色紧急模式
3. 日志显示辐射联动信息
4. 医疗安全性上升，食物稳定性下降

---

### 4. 食物联动系统
**触发条件**: 食物稳定性 < 30%
**联动效果**:
- 自动启动紧急营养分配方案
- 宇航员配给减少（模拟 crew_count - 1）
- 食谱调整为"紧急配给模式：每日热量摄入降低20%"
- 日志显示："食物联动：已启动紧急营养分配方案"

**验证方法**:
1. 长时间运行让食物自然衰减
2. 切换到"能源管理"视图
3. 看到食谱建议变为紧急配给模式

---

### 5. 紧急协议模式
**触发条件**: 
- 辐射等级 > 80，或
- 能源水平 < 10%

**联动效果**:
- UI 整体变红（CSS emergency-mode）
- 全屏红色警报扫描动画（GSAP）
- 标题呼吸灯闪烁（0.5s 周期）
- AI 自动执行资源保全策略

**验证方法**:
1. 等待极端事件触发
2. 观察整个界面变为红色调
3. 顶部标题开始闪烁红光

---

### 6. 多系统数据实时同步
**前端联动**:
- ✓ 顶部状态栏实时更新：宇航员人数、基地稳定率
- ✓ 预计生存时间动态计算：生存指数 × 1.8

## 🔧 技术亮点总结

1. **真实 AI 驱动**: 不是预设脚本，每次决策都调用智谱 GLM-4-AIR
2. **双向联动**: AI 建议 → 解析关键词 → 执行真实数值变化 → 反馈到 UI
3. **多层应急**: 本地规则联动 + 云端 AI 决策 + 紧急协议三重保障
4. **可视化解释**: 每条 AI 决策都有颜色编码的执行记录
5. **航天级 UI**: Three.js 星空 + GSAP 动画 + ECharts 数据可视化
